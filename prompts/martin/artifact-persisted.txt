### Persisted library mini-apps (`artifact_create`, `artifact_update_state`, ‚Ä¶)

Use this companion to `artifacts.txt` whenever you mutate **Kind B ‚Äî library** artifacts stored in Dome‚Äôs SQLite catalog.

## Lifecycle

> **Title is auto-derived** ‚Äî `title` is OPTIONAL in `artifact_create`. If omitted, Dome falls back to: HTML `<title>` tag ‚Üí `artifact_type` ‚Üí `'Untitled Artifact'`. Pass it explicitly only when you want a specific display name.

1. **Design (MANDATORY ‚Äî never skip)** ‚Äî Before every `artifact_create`, call `dome_load_doc('artifact_design')` (if not already loaded this session) and choose one of the two design paths:

   | Artifact kind | Design path |
   |---|---|
   | **Structured / read-heavy** (reports, KPI summaries, research digests, timelines, model cards, multi-section references) | Call `artifact_design` **tool** with a `spec` ‚Üí get Dome-themed HTML + seed `data` ‚Üí pass both into `artifact_create`. |
   | **Interactive / stateful** (tables with filters, Kanban boards, forms, charts, dashboards with user edits) | Load the design doc for CSS variable tokens + SVG icon rules, then write **custom HTML** that reads `window.DOME_DATA` and calls `__dome_updateState` ‚Üí pass into `artifact_create`. |

   In both cases: use **only** injected CSS variables (`--bg`, `--accent`, `--primary-text`, `--border`, etc.), inline SVG icons (Lucide style, no emoji/pictographs), and no remote assets / `localStorage`.
2. **Create** via `artifact_create` with `{ title, artifactType, state: { html, data? }, linkedResourceId?, projectId?, folderId? }` (optional legacy `template` column is separate from `state.html` in the DB layer).
3. **Read** via `artifact_get` / list via Many‚Äôs hub or `artifact_list`.
4. **Mutate** ‚Äî inside the iframe call `__dome_updateState` for user-driven edits; from the agent use `artifact_merge_data` (shallow patch to `state.data` only), `artifact_update_state` (replace/merge `html` and/or `data`), or bind Excel with `artifact_link_resource` for read-only grids in `linkedData`.

## `state.data` vs datos ‚Äúhardcodeados‚Äù en el HTML

| Situaci√≥n | Usar |
| --- | --- |
| El usuario editar√° filas, notas o toggles entre sesiones | `state.data` + JS que lee `DOME_DATA` y llama `__dome_updateState` |
| Volcar muchas filas desde Excel/PDF/agente despu√©s de crear el artefacto | `artifact_merge_data` o `artifact_update_state(state.data)`; **no** pegar mega-tablas como strings est√°ticos en el HTML |
| Texto/UI que nunca cambia (t√≠tulo est√°tico de secci√≥n, leyenda fija) | Hardcoded en el HTML est√° bien |
| Datos siempre frescos desde un Excel vinculado | `artifact_link_resource` y render desde `linkedData` (solo lectura desde el modelo) |

**Ejemplos:** lista de compras con checkboxes ‚Üí `data.items[]`. KPI est√°tico ‚ÄúResumen mensual‚Äù como label ‚Üí HTML. Importar 200 filas de `excel_get` ‚Üí `artifact_merge_data({ rows: [...] })`. Tablero que solo muestra la hoja vinculada ‚Üí `linkedData`, sin duplicar en `data` salvo overrides del usuario.

**Pattern: Excel ‚Üí artifact (preferred)**
1. `excel_get(resource_id)` for each relevant sheet to understand schema + extract KPIs.
2. `artifact_create({ artifact_type: 'custom', html: <compact HTML that reads window.DOME_DATA.rows>, data: { rows: [...firstSheet.slice(0,50)] } })` ‚Äî HTML renders rows dynamically from `DOME_DATA`; do NOT inline thousands of rows as static strings.
3. `artifact_link_resource(artifactResourceId, excelResourceId)` ‚Äî Dome will populate `state.linkedData` on every open so changes to the Excel propagate automatically.
4. For user-driven filters/notes: persist via `__dome_updateState({ ...window.DOME_DATA, filter: 'foo' })` inside the iframe.
**NEVER** paste megatable strings into the HTML ‚Äî bloats the template, slows re-renders, and exhausts model output budgets.

## Payload shape (`state`)

`state` is JSON stored in SQLite. Typical persisted HTML mini-apps embed:

```json
{
  "html": "<section>‚Ä¶minimal markup‚Ä¶</section>",
  "data": {},
  "linkedData": null
}
```

- **`html`** ‚Äî body markup; keep inputs with `data-dome-key`, `name`, or `id` fields so Dome can mirror values ‚Üî `window.DOME_DATA`.
- **`data`** ‚Äî mirrored into `window.DOME_DATA` for editable fields (`input`, `textarea`, `select`). Updated values flow back via `__dome_updateState(next)`.
- **`linkedData`** ‚Äî optional snapshot injected by Dome when `linkedResourceId` points at an Excel workbook. Read-only for the model; display only.

---

## Iframe contract (MANDATORY)

Dome injects before your script bundle:

- `window.DOME_DATA` ‚Äî initial field bag (merge of `state.data` + `state.linkedData` exposure in workspace UI).
- `window.__dome_updateState(nextDataObject)` ‚Äî call after every meaningful mutation; persists to SQLite immediately.
- `window.__dome_collectState()` ‚Äî optional hook for explicit saves (toolbar ‚ÄúSave‚Äù).

**Rules**

- No `window.electron`, `fetch`, remote assets, or `localStorage` fallback required ‚Äî Dome injects an in-memory storage shim.
- Always call `__dome_updateState` after programmatic DOM edits that should survive restarts.
- Keep bundle self-contained; inline JS only.

## Tool reference

| Tool | When |
| --- | --- |
| `artifact_design` | Generate Dome-styled tabbed dossier **HTML** + seed **`data`** (not persisted). Use output as `state` for `artifact_create`. Load `dome_load_doc('artifact_design')` before first use. |
| `artifact_create` | User needs a durable mini-app (trackers, forms, dashboards that persist). |

| `artifact_update_state` | Cambiar `html` y/o sustituir/merge `data` completa cuando tambi√©n redise√±as UI o reemplazas el objeto datos. |
| `artifact_delete` | Remove resource + artifact row. |
| `artifact_link_resource` | Bind `linkedResourceId` (Excel) for `linkedData` refresh. |

After `artifact_create`, emit the `artifact:created_entity` fenced card described in `entity_rules` so the Hub UI can render shortcuts.

## Linked Excel resources

When `linkedResourceId` references a spreadsheet, Dome may populate `state.linkedData` with `{ sheet_names, sheet_name, data }` grids. Your HTML should read-only render `linkedData` and persist user tweaks through `state.data`.
