### Persisted library mini-apps (`artifact_create`, `artifact_update_state`, …)

Use this companion to `artifacts.txt` whenever you mutate **Kind B — library** artifacts stored in Dome’s SQLite catalog.

---

## Lifecycle

1. **Design (optional)** — For tabbed dossiers, call `artifact_design` **before** `artifact_create`. It returns `html` plus `data` with at least `activeTab` and `layoutKind: "dome-design-v1"`. Tab clicks call `__dome_updateState` and persist `activeTab`.
2. **Create** via `artifact_create` with `{ title, artifactType, state: { html, data? }, linkedResourceId?, projectId?, folderId? }` (optional legacy `template` column is separate from `state.html` in the DB layer).
3. **Read** via `artifact_get` / list via Many’s hub or `artifact_list`.
4. **Mutate** — inside the iframe call `__dome_updateState` for user-driven edits; from the agent use `artifact_merge_data` (shallow patch to `state.data` only), `artifact_update_state` (replace/merge `html` and/or `data`), or bind Excel with `artifact_link_resource` for read-only grids in `linkedData`.

---

## `state.data` vs datos “hardcodeados” en el HTML

| Situación | Usar |
| --- | --- |
| El usuario editará filas, notas o toggles entre sesiones | `state.data` + JS que lee `DOME_DATA` y llama `__dome_updateState` |
| Volcar muchas filas desde Excel/PDF/agente después de crear el artefacto | `artifact_merge_data` o `artifact_update_state(state.data)`; **no** pegar mega-tablas como strings estáticos en el HTML |
| Texto/UI que nunca cambia (título estático de sección, leyenda fija) | Hardcoded en el HTML está bien |
| Datos siempre frescos desde un Excel vinculado | `artifact_link_resource` y render desde `linkedData` (solo lectura desde el modelo) |

**Ejemplos:** lista de compras con checkboxes → `data.items[]`. KPI estático “Resumen mensual” como label → HTML. Importar 200 filas de `excel_get` → `artifact_merge_data({ rows: [...] })`. Tablero que solo muestra la hoja vinculada → `linkedData`, sin duplicar en `data` salvo overrides del usuario.

## Payload shape (`state`)

`state` is JSON stored in SQLite. Typical persisted HTML mini-apps embed:

```json
{
  "html": "<section>…minimal markup…</section>",
  "data": {},
  "linkedData": null
}
```

- **`html`** — body markup; keep inputs with `data-dome-key`, `name`, or `id` fields so Dome can mirror values ↔ `window.DOME_DATA`.
- **`data`** — mirrored into `window.DOME_DATA` for editable fields (`input`, `textarea`, `select`). Updated values flow back via `__dome_updateState(next)`.
- **`linkedData`** — optional snapshot injected by Dome when `linkedResourceId` points at an Excel workbook. Read-only for the model; display only.

---

## Iframe contract (MANDATORY)

Dome injects before your script bundle:

- `window.DOME_DATA` — initial field bag (merge of `state.data` + `state.linkedData` exposure in workspace UI).
- `window.__dome_updateState(nextDataObject)` — call after every meaningful mutation; persists to SQLite immediately.
- `window.__dome_collectState()` — optional hook for explicit saves (toolbar “Save”).

**Rules**

- No `window.electron`, `fetch`, remote assets, or `localStorage` fallback required — Dome injects an in-memory storage shim.
- Always call `__dome_updateState` after programmatic DOM edits that should survive restarts.
- Keep bundle self-contained; inline JS only.

## Tool reference

| Tool | When |
| --- | --- |
| `artifact_design` | Generate Dome-styled tabbed dossier **HTML** + seed **`data`** (not persisted). Use output as `state` for `artifact_create`. Load `dome_load_doc('artifact_design')` before first use. |
| `artifact_create` | User needs a durable mini-app (trackers, forms, dashboards that persist). |
| `artifact_get` | Need current `state` / metadata. |
| `artifact_merge_data` | Solo actualizar `state.data` por lotes (p. ej. tras `excel_get` / extracción) sin re-enviar HTML. Merge superficial por claves de primer nivel. |
| `artifact_update_state` | Cambiar `html` y/o sustituir/merge `data` completa cuando también rediseñas UI o reemplazas el objeto datos. |
| `artifact_delete` | Remove resource + artifact row. |
| `artifact_link_resource` | Bind `linkedResourceId` (Excel) for `linkedData` refresh. |

After `artifact_create`, emit the `artifact:created_entity` fenced card described in `entity_rules` so the Hub UI can render shortcuts.

---

## Linked Excel resources

When `linkedResourceId` references a spreadsheet, Dome may populate `state.linkedData` with `{ sheet_names, sheet_name, data }` grids. Your HTML should read-only render `linkedData` and persist user tweaks through `state.data`.
