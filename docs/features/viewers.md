# Viewers Feature

Documentation for Dome's resource viewers: PDF, Video, Audio, Image, URL, Notebook. Renderer components live in `app/components/viewers/`; PDF has its own `pdf/` subdirectory for page and annotation rendering. Main-process services live under `electron/documents/`, `electron/feeders/`, and `electron/transcription/`, with IPC handlers in `electron/ipc/media/`.

---

## Interfaces

### Viewer usage

- **Switch** (`app/components/workspace/WorkspaceLayout.tsx`): chooses viewer by `resource.type` — `pdf` → `PDFViewer`, `video` → `VideoPlayer`, `audio` → `AudioPlayer`, `image` → `ImageViewer`, `url` → `URLViewer`.
- **Notebook** is not a viewer in this directory: it is mounted as its own workspace route at `app/workspace/notebook/[[...params]]/client.tsx` (`NotebookEditor` + `PyodideProvider`).
- **Resource**: must have `id`; for binary types often `internal_path` / `file_path` (resolved via `window.electron.resource.getFilePath`); URL type has `metadata?.url`.
- **Data source**: file content via `window.electron.resource.readFile(resourceId)` (Base64 data URL) — used by image / video / audio and the PDF blob URL path.

### PDFViewer

- **Role**: render PDF pages, annotations overlay, navigation (page input, prev/next, zoom), optional outline, region-select handoff to AI, open-external.
- **Components**:
  - `app/components/viewers/PDFViewer.tsx` — container (toolbar, page state, annotation state, outline, region mode).
  - `app/components/viewers/pdf/PDFPage.tsx` — single page canvas; calls `window.electron.pdf.renderPage(...)` for rasterization.
  - `app/components/viewers/pdf/AnnotationLayer.tsx` — annotation overlay (renders + interacts).
  - `app/components/viewers/pdf/PDFHighlightsList.tsx` / `PDFNotesList.tsx` — annotation side panels.
  - `app/components/viewers/pdf/PDFOutline.tsx`, `PDFThumbnails.tsx`, `PDFPageInput.tsx`, `PDFViewerSkeleton.tsx` — auxiliary UI.
- **Annotations**: stored as `resource_interactions` with `type='annotation'` and `position_data` (page, rect, etc.). Loaded via `useInteractions(resource.id)`; helpers in `app/lib/pdf/annotation-utils.ts`.
- **Libs**: `app/lib/pdf/pdf-loader.ts` (PDF.js document load, page, outline); `app/lib/pdf/annotation-utils.ts` (parse/serialize); `app/lib/pdf/pdf-region-handoff.ts` (region → AI handoff).
- **Main process**: PDF page rendering is invoked from the renderer via the `pdf:render-page` IPC channel — handler in `electron/ipc/media/pdf-render.cjs`. Thumbnails (first-page) for the library are generated by `electron/documents/thumbnail.cjs` (PDF branch).

### VideoPlayer / AudioPlayer

- **Role**: play video or audio from the resource file. Source = blob URL or data URL from `window.electron.resource.readFile(resourceId)`.
- **Metadata**: `duration`, `transcription` in `resource.metadata`; optional AI summary.
- **YouTube ingest**: `electron/feeders/youtube-service.cjs` extracts video id / metadata when a YouTube URL is added as a resource (used during ingestion, not at playback).
- **Transcription**: `electron/transcription/transcription-service.cjs` produces transcripts consumed by AudioPlayer / VideoPlayer metadata; session / permission / settings channels live in `electron/ipc/media/transcription.cjs` (`transcription:get-settings`, `transcription:session-start`, `transcription:resource-to-note`, …).

### ImageViewer

- **Role**: display image from the resource (data URL from `readFile` or blob).
- **Thumbnails**: previews for the resource library are generated by `electron/documents/thumbnail.cjs` (image branch, sharp). Image-specific transforms (crop / resize / metadata) are exposed via `electron/ipc/media/images.cjs` (`image:crop`, `image:resize`, `image:thumbnail`, `image:metadata`).

### URLViewer

- **Role**: display URL resources — embed in iframe or render scraped content. Metadata distinguishes `url_type: article | youtube`.
- **Web processing**: `app/lib/web/processor.ts` plus the main-process scraper `electron/feeders/web-scraper.cjs` (which delegates to `services/web/fetch-dispatcher.cjs` and `services/web/search-dispatcher.cjs`). Exposed to the renderer as the IPC channel `web:scrape` (`window.electron.web.scrape(...)`).

### Notebook (separate workspace route, not in `app/components/viewers/`)

- **Renderer**: `app/workspace/notebook/[[...params]]/client.tsx` mounts `NotebookEditor` with `PyodideProvider`. Notebook resources are routed here instead of the viewer switch.
- **Main process**: `electron/documents/notebook-python.cjs` runs Python cells / manages venvs. IPC handlers in `electron/ipc/media/notebook.cjs` — `notebook:runPython`, `notebook:checkPython`, `notebook:createVenv`, `notebook:pipInstall`, `notebook:checkVenv`, `notebook:pipList`, `notebook:pipInstallFromRequirements`.
- **Metadata**: `notebook_workspace_path`, `notebook_venv_path` on `resource.metadata`.

## Design patterns

### Loading file content

- **Renderer**: call `window.electron.resource.readFile(resourceId)` → returns a Base64 data URL. Viewer creates a blob URL (or uses the data URL directly) as the media `src`.
- **Lazy load**: viewers mount only when the tab/resource is active; file load happens in `useEffect` keyed on `resourceId`. In `WorkspaceLayout`, viewers are imported with `React.lazy(...)`.

### PDF-specific

- **pdf-loader**: load PDF.js document from a URL or buffer; expose `getPage(pageNumber)`, `numPages`, outline.
- **PDFPage**: renders one page on a canvas. Calls `window.electron.pdf.renderPage({ filePath, pageNumber, scale, format })` → IPC `pdf:render-page` (handler in `electron/ipc/media/pdf-render.cjs`).
- **AnnotationLayer**: renders annotation shapes from `position_data`; supports creation flows that call `db.interactions.create({ resourceId, type: 'annotation', content, position_data })`.
- **Region handoff**: `pdf-region-handoff.ts` converts a selected rect into the AI chat payload used by the "ask about this" feature.

### Annotations (interactions)

- **Type**: `resource_interactions` with `type='annotation'`.
- **position_data**: JSON `{ page, rect, selectedText, shape, … }`.
- **Load / mutate**: `useInteractions(resource.id)` returns `{ annotations, addInteraction, updateInteraction, deleteInteraction }`.

### Events

- **resource:updated**: viewers can subscribe to refresh when the resource or its metadata (e.g. transcription) changes.

## Data flow

- **Open resource**: `WorkspaceLayout` matches `resource.type` and lazy-mounts the right viewer with the `Resource`. The viewer's `useEffect` calls `window.electron.resource.readFile(resourceId)` → sets `src` state → renders.
- **PDF**: `readFile` → `loadPDFDocument` → `numPages` → render `PDFPage` per page (canvas via `pdf:render-page`); `AnnotationLayer` reads interactions for the resource and filters by page; outline / thumbnails fetched via the PDF.js doc.
- **Create annotation**: user selects text in PDF → toolbar action → build `position_data` → `db.interactions.create(...)` → refresh annotations.
- **URL**: `URLViewer` reads `resource.metadata?.url` or `resource.content`; either embeds the URL in an iframe or calls `web:scrape` to fetch scraped content.
- **Notebook**: route match → `NotebookEditor` → cell execution goes through `notebook:runPython` (handler in `electron/ipc/media/notebook.cjs`, backed by `electron/documents/notebook-python.cjs`).

## Functionality

- **PDF**: multi-page view, zoom (fit-width / fit-page / custom), page navigation, annotation overlay (create/list/delete), outline, thumbnails, region-select → AI handoff, open-external.
- **Video**: playback from resource file; thumbnails via `electron/documents/thumbnail.cjs` (video branch, ffmpeg); YouTube metadata via `electron/feeders/youtube-service.cjs`; transcription in `resource.metadata`.
- **Audio**: playback from resource file; transcript display from `resource.metadata.transcription` (produced by `electron/transcription/transcription-service.cjs`).
- **Image**: display image; thumbnails via `electron/documents/thumbnail.cjs` (image branch).
- **URL**: embed in iframe or render scraped content; YouTube / article metadata; channel `web:scrape`.
- **Notebook**: Python cell execution, venv management, pip install — IPC `notebook:runPython` and friends, backed by `electron/documents/notebook-python.cjs`.
