# Draft de Video Demo para Dome

## Resumen ejecutivo

Este draft propone una demo de `3:30` enfocada en una audiencia generalista: investigadores, estudiantes avanzados, founders y knowledge workers. La idea es mostrar a Dome como un workspace donde la IA no solo responde preguntas, sino que entiende recursos reales y los transforma en resultados utilizables.

Flujo principal recomendado:

`URL/YouTube -> Listo para IA -> Many -> Studio`

## Mensaje central

`Dome convierte recursos dispersos en un workspace donde la IA puede leer, razonar y producir artefactos útiles.`

## Qué queremos que la audiencia entienda

1. Dome centraliza distintos tipos de conocimiento en un solo lugar.
2. El contenido no se queda almacenado: queda `Listo para IA`.
3. `Many` trabaja con contexto real del recurso actual.
4. Dome puede transformar lectura en outputs útiles como `mindmaps`, `quizzes` o `flashcards`.
5. La plataforma tiene profundidad futura con `MCP`, agentes y equipos de agentes, aunque no hace falta mostrarlo todo en una sola pieza.

## Storyline recomendada

### Escena 1. Hook

**Tiempo:** `0:00-0:20`

**Qué se ve en pantalla**

- Proyecto abierto con mezcla de recursos: PDF, URL, video o PowerPoint.
- Paneo rápido por biblioteca o workspace.

**Voz en off**

`Tus PDFs, URLs, videos y notas ya no tienen que vivir separados. En Dome, todo tu conocimiento queda en un solo workspace y listo para trabajar con IA.`

**Objetivo narrativo**

Posicionar a Dome desde el primer segundo como una capa de trabajo sobre conocimiento, no como una simple app de notas.

### Escena 2. Ingesta simple, resultado potente

**Tiempo:** `0:20-0:50`

**Qué se ve en pantalla**

- Abrir `Command Center`.
- Pegar una URL o un enlace de YouTube.
- Crear el recurso y abrirlo dentro del proyecto.
- Esperar el estado `Listo para IA`.

**Voz en off**

`Agregar conocimiento debería ser tan simple como pegar un enlace. Dome captura ese contenido, lo incorpora al proyecto y lo prepara para que la IA pueda entenderlo de verdad.`

**Momento clave**

Detenerse un segundo en el badge `Listo para IA`. Ese microdetalle comunica preparación real del contenido.

### Escena 3. Many como copiloto contextual

**Tiempo:** `0:50-1:45`

**Qué se ve en pantalla**

- Abrir `Many` mientras el recurso está visible.
- Escribir un prompt corto y claro.
- Mostrar streaming y tool cards.
- Resaltar que la respuesta sale del recurso actual, no de una pregunta abstracta.

**Voz en off**

`Aquí es donde Dome cambia de categoría. Many no está conversando sobre una caja vacía: entiende el recurso que estoy viendo, puede usar mis recursos, buscar en web y devolverme una respuesta aterrizada en mi contexto.`

**Prompt recomendado**

`Resume este recurso en 5 ideas clave y dime por qué importa.`

**Prompts alternativos**

- `Extrae los conceptos principales y conviértelos en una guía breve para estudio.`
- `¿Qué partes de este documento son más útiles para preparar una presentación?`
- `Compara este recurso con lo que ya tengo en el proyecto y detecta temas repetidos o complementarios.`

### Escena 4. De lectura a output útil

**Tiempo:** `1:45-2:35`

**Qué se ve en pantalla**

- Ir a `Studio`.
- Elegir el proyecto actual.
- Generar un `Mindmap` o `Flashcards`.
- Abrir el output resultante.

**Voz en off**

`La diferencia no es solo que Dome responde preguntas. También convierte lo que lees en algo que puedes usar: una estructura visual, material de estudio o una guía accionable.`

**Opción principal**

`Mindmap`

Es el output más visual y más fácil de entender en pocos segundos.

**Opción alternativa**

`Flashcards`

Funciona muy bien si la demo apunta a educación, estudio o investigación académica.

### Escena 5. Capa avanzada de plataforma

**Tiempo:** `2:35-3:05`

**Qué se ve en pantalla**

- Corte breve a `MCP Settings`, `Many Agents` o `Agent Team`.
- No profundizar demasiado: basta con una señal clara de extensibilidad.

**Voz en off**

`Y si quieres ir más allá, Dome no se queda en un solo asistente. Puedes conectarlo a herramientas externas con MCP o crear agentes especializados para distintos tipos de trabajo.`

**Recomendación**

Para una demo generalista, usar `MCP` como teaser rápido porque comunica plataforma sin abrir demasiados frentes.

### Escena 6. Cierre

**Tiempo:** `3:05-3:30`

**Qué se ve en pantalla**

- Montaje rápido: recurso original, badge `Listo para IA`, chat de `Many`, output de `Studio`.
- Cerrar con una vista limpia del producto.

**Voz en off**

`Dome no solo almacena conocimiento. Lo convierte en algo que puedes consultar, organizar y transformar.`

## Prompts concretos para usar dentro de Dome

### Prompt principal

`Resume este recurso en 5 ideas clave y dime por qué importa.`

### Prompts alternativos

- `Extrae los conceptos principales y conviértelos en una guía breve para estudio.`
- `¿Qué partes de este documento son más útiles para preparar una presentación?`
- `Genera flashcards a partir de este recurso con foco en conceptos difíciles.`
- `Compara este recurso con lo que ya tengo en el proyecto y detecta temas repetidos o complementarios.`
- `Convierte este contenido en una estructura para un mindmap.`

## Cosas interesantes que conviene mostrar

- El badge `Listo para IA`, porque hace tangible la preparación del contenido.
- Que `Many` trabaja sobre el recurso actual y no sobre una pantalla vacía.
- El streaming de respuesta y las tool cards, porque muestran trabajo real.
- La transición de input bruto a output útil: recurso a `mindmap`, `quiz` o `flashcards`.
- La variedad de recursos soportados: PDF, URL, YouTube y PowerPoint.
- `MCP` como señal de extensibilidad de producto.
- `Many Agents` o `Agent Team` como teaser para una audiencia más técnica.

## Qué no conviene meter en esta demo

- Demasiadas features en una sola pieza.
- Explicaciones muy técnicas sobre arquitectura o LangGraph.
- Un flujo largo de configuración antes de mostrar valor.
- Varias ramas narrativas a la vez.

## Edición y ritmo

- Usar cortes rápidos en la primera mitad para instalar la promesa.
- Dejar respiración visual cuando aparezca `Listo para IA`.
- Dar más tiempo a `Many` y `Studio` que a settings.
- Evitar overlays excesivos: el producto ya es visualmente suficiente.

## Variantes por audiencia

### Generalista

Usar el flujo principal de este documento: `URL/YouTube -> Listo para IA -> Many -> Studio`.

### Académica

Cambiar el recurso inicial por un PDF y cerrar con `Flashcards` o `Guide`.

### Técnica

Mantener `Many`, pero sustituir `Studio` por `MCP` o `Agent Team`.
