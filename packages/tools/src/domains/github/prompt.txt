# GitHub Seguimiento tools — operational guide

Load this doc (`dome_load_doc({ id: "github_tool" })`) before mutating GitHub (`github_create_issue`, `github_create_milestone`, `github_update_issue`).

## Sync model
- Dome mirrors selected repos from **GitHub → Seguimiento** (SQLite + calendar projection).
- **`repo_id`**: Dome id from `github_list_repos` (e.g. `ghr-…`), **not** `owner/repo` string.
- **`issue_id`**: Dome id from `github_list_issues` (e.g. `ghi-…`).
- Writes go to GitHub API then refresh local store; call `github_sync` if UI seems stale.

## Read tools
| Tool | Use when |
|------|----------|
| `github_list_repos` | Need `repo_id`; `selected_only` default true |
| `github_upcoming_milestones` | Cross-repo delivery dates / roadmap |
| `github_list_milestones` | Milestones for one repo |
| `github_list_issues` | Issues for one repo; filter `state` |
| `github_sync` | Force full sync now |

## Write tools (GitHub API)
| Tool | Required fields |
|------|-----------------|
| `github_create_issue` | `repo_id`, `title`; optional `body`, `milestone_number`, `labels`, **`assignees`** (GitHub logins) |
| `github_update_issue` | `issue_id`; patch `title`, `body`, `state`, `milestone_number` |
| `github_create_milestone` | `repo_id`, `title`; optional `description`, `due_on` (ISO date), `state` open/closed |

## Mentioned people → assignees
- When Source has **mentioned-people** with `github:login`, pass those logins in `assignees` (no `@`).
- Do **not** invent logins. If no github identity, ask once or note “suggested assignee” in the body.
- If assign fails for permissions, keep the issue and mention the suggested assignee in the body.

## HITL (Many only)
Creating/updating issues and milestones requires **user approval** in Many.
Workflows and automations run with `skipHitl` — no approval card.

## Tips
- Always `github_list_repos` before first write if you do not have `repo_id`.
- Milestone numbers are per-repo GitHub numbers, not Dome ids.
- Source tag in results: `"source": "github"`.
- Light `remember_fact` (general) is OK after creating an assigned issue; do not dump issue bodies into MEMORY.
