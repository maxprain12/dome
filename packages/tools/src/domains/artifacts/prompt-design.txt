# Artifact design — tabbed dossier generator (`artifact_design` tool)

Use for **read-heavy structured artifacts**: multi-section reports, research digests, model cards, benchmarks, timelines. The tool turns a JSON `spec` into Dome-themed HTML (header, subtitle, navigable tabs, card sections with kickers/badges) plus seed `data` — you then persist it with `artifact_create`.

For **interactive/stateful** artifacts (editable tables, boards, forms) skip this tool and hand-write data-driven HTML per the `artifact_persisted` doc.

## Flow

1. `dome_load_doc('artifact_design')` once per session (this doc).
2. Call `artifact_design` with one argument: `spec` (schema below).
3. Optionally post-process the returned HTML (e.g. prepend inline SVG icons to tab labels).
4. `artifact_create` with `artifact_type: "custom"`, `html: <returned html>`, `data: <returned data>` (merge extra keys into `data` first if you need them).
5. Emit `artifact:created_entity` per `entity_rules`.

The tool does NOT persist anything — it only builds `html` + `data` seeds (`activeTab`, `layoutKind`). Tab selection persists at runtime via `__dome_updateState`.

## `spec` schema

- **title** (string, required) — main heading, plain text.
- **subtitle** (string, optional) — line under the title.
- **active_tab** (string, optional) — initial tab id (defaults to first).
- **tabs** (array, required) — `{ "id": "slug", "label": "Plain text" }`. Ids: letters/digits/`_`/`-` only, stable (`overview`, `benchmarks`…). Labels: NO emoji.
- **panels** (object, required) — one key per tab id → `{ "sections": [ … ] }`.

**Section:** `{ kicker?, badge?, badge_tone?, blocks }`
- `kicker` — small-caps row label (e.g. `KEY FINDINGS`).
- `badge` — short pill (e.g. `NOVEL`); `badge_tone`: `neutral|info|success|warning|error`.
- `blocks` — ordered content:
  - `{ "type": "paragraph", "text": "…" }`
  - `{ "type": "numbered", "number": 1, "title": "…", "body": "…", "bullets": ["…"]? }`
  - `{ "type": "bullets", "items": ["…"] }`
  - `{ "type": "code", "text": "…" }` (monospace, `var(--font-mono)`)

Keep copy concise — oversized specs are clipped server-side. Split long content across tabs instead of one giant panel.

## Hard rules

- **Icons:** never emoji/pictographs/OS-font glyphs. Inline SVG only, Lucide style: `viewBox="0 0 24 24"`, `fill="none"`, `stroke="currentColor"`, `stroke-width="2"`, round caps/joins, 16–20px, `aria-hidden="true"` when decorative.
- **Language:** all user-visible strings in the user's conversation language. No CJK unless explicitly requested.
- **Styling:** the generator emits only injected CSS variables (`--bg`, `--bg-secondary`, `--primary-text`, `--accent`, `--border`, semantic tokens, `--radius-*`, `--space-*`, `--font-sans`, `--font-mono`). Keep any HTML you add on the same tokens — no hex, no gradients, no `prefers-color-scheme`.
- **Runtime:** no `localStorage`/`sessionStorage`/remote assets; persistence goes through `__dome_updateState` (see `artifact_persisted`).

## Extending after the scaffold

Need inputs, filters, or automations on top of the dossier? Extend the returned `data` and HTML before `artifact_create` — keep the body-fragment rule, render additions from `window.DOME_DATA`, and call `__dome_updateState` after every mutation.
